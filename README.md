Originally intended to be HSL, the redundant lightness/darkness (see hexcone vs double hexcone) led to HSV being chosen.

Updated Plugin/SSTUTools/KSPSHaderTools files to support HSV input on the frontend while preserving the RGB pipeline in part configs and rendering. Includes a toggle to convert between RGB and HSV input methods with automatic conversion and sync between.
